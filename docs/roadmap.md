# Mimir Roadmap

> A plastic memory well for coding agents.

## 一、当前状态

本项目命名为 **Mimir**，取自北欧神话中守护智慧之泉的巨人。目标是构建一个可随使用场景持续演化的本地记忆层。现有实现已收敛到一条简洁路线：**Mimir，其核心实现为原型可塑网络（Prototypical Plastic Network, PPN）**。

---

## 二、核心架构方向：PPN

### 2.1 核心洞察

Mimir 的隐喻是智慧之井——知识在使用过程中不断沉淀、演化，并可被持续汲取。在系统中，这意味着：

- 记忆不是离散事件的堆砌；
- 而是可随经验持续演化的参数化表征；
- 最终以可持久化、可解释、可度量的形式存在。

原有探索方向曾考虑过快慢权重分离、记忆痕迹（MemoryTrace）、语义簇中心等结构，但它们带来两个关键问题：

1. **推理与学习耦合**：`encode()` 操作中嵌入聚类、记忆更新、维护任务等副作用，导致推理延迟不可预测，也违反了读写分离（CQRS）原则。
2. **状态过度碎片化**：多个子系统各自维护一份状态，彼此一致性代价高，调试困难。

### 2.2 大胆简化

**用单一数据结构承载全部可塑性：一个定长原型向量矩阵。**

```text
快权重 = [p₁, p₂, ..., pₖ]  其中每个 pᵢ ∈ ℝᵈⁱᵐ
```

每个原型 `pᵢ` 代表一个被系统反复激活的语义概念。整个 Mimir 系统退化为：

- **推理**：输入 base embedding 与所有原型计算相似度，加权残差调制输出。
- **预测**：根据前一步激活的原型，预测下一步应该激活哪个原型（预测编码 / Predictive Coding）。
- **学习**：新输入激活最接近的原型，并向其方向微调；预测命中时强化转移，预测错误时弱化转移。
- **遗忘**：原型强度指数衰减 + 容量满时淘汰弱原型。
- **持久化**：只保存 `[k × dim]` 原型矩阵 + `[k × k]` 转移矩阵 + 元数据。

这一设计与心理学中的**原型理论（Prototype Theory）**一致：人类记忆不是离散事件的堆砌，而是围绕典型范例形成的压缩表征。同时，系统引入**预测编码（Predictive Coding）**机制——大脑不是被动接收输入，而是持续生成预测，只把"预测错误"当作值得学习的新信号。

### 2.3 理论基础

本架构并非凭空设计，而是建立在一系列经充分验证的神经科学与计算记忆研究之上。以下论文与理论为关键模块提供了直接支撑：

#### 2.3.1 突触可塑性：Oja 与 BCM 规则

- **Hebb 规则（1949）**："同时放电，同步连接"是局部学习的基本假设，但原始规则会导致权重无限增长。
- **Oja 规则（1982）**：在 Hebb 基础上加入权重归一化，收敛后指向数据协方差矩阵的第一主成分。数学稳定，适合作为基线学习规则。
- **BCM 规则（1982）**：引入滑动阈值，实现双向可塑性——突触后活动高于阈值时增强（LTP），低于阈值时抑制（LTD）。这提供了"熟悉后收益递减"的自然机制，比 Oja 更适合选择性记忆与遗忘。

**对 Mimir 的启示**：P0-P1 使用 Oja 规则保证稳定基线；P2 引入 BCM 作为可选策略，实现更丰富的增强/抑制动态。

#### 2.3.2 快慢权重分离

- **Hinton & Plaut（1987）**：首次提出双权重思想——慢权重存储长期知识，快权重存储临时知识，以缓解灾难性遗忘。
- **Ba & Hinton（2016）**：提出通过外积累积更新快权重矩阵：
  $$A(t) = \lambda A(t-1) + \eta h(t)h(t)^T$$
  其中 $\lambda$ 是衰减因子，$\eta$ 是学习率。快权重提供短期记忆的"软注意力"机制。

**对 Mimir 的启示**：原型矩阵的更新可直接采用外积 + 衰减形式；预测模块的转移矩阵也可沿用同一机制。

#### 2.3.3 海马体启发架构

- **DG（齿状回）**执行**模式分离**：通过稀疏激活（约 3-5%）将相似输入映射到几乎不重叠的表征。
- **CA3** 是自联想记忆网络，实现**模式完成**：即使输入线索不完整，也能重构完整表征。
- **CA1** 整合 CA3 完成模式与皮层输入，参与记忆巩固。
- **HiCL（AAAI 2026）**：将 DG-CA3-CA1 三突触回路完整映射到深度学习，提出 DG 门控的混合专家路由。

**对 Mimir 的启示**：原型激活应从 softmax 全参与改为 **top-k 稀疏激活**（如 k=5%），更接近 DG 的稀疏编码，也更高效、更可解释。

#### 2.3.4 互补学习系统与记忆巩固

- **CLS 理论（McClelland et al., 1995）**：海马体快速学习情景记忆，新皮层慢速提取语义记忆。巩固通过睡眠期间的**重放（replay）**完成。
- **睡眠重放证据**：海马体位置细胞在慢波睡眠中以压缩时间尺度（快 10-20 倍）共同重激活，与皮层纺锤波耦合促进信息转移。

**对 Mimir 的启示**：`consolidate()` 不应只是强化强度，未来应实现基于 surprise 的优先重放机制，将高显著性记忆巩固为长期原型。

#### 2.3.5 现代 Hopfield 网络

- **Ramsauer et al.（2020）**：证明现代连续 Hopfield 网络的更新规则在数学上等价于 Transformer 注意力机制，存储容量随维度指数增长。

**对 Mimir 的启示**：原型矩阵本质上就是一个 Hopfield 联想记忆。可以用内容寻址视角理解 `lookup()`：输入 embedding 作为查询，原型作为存储模式，返回最相似模式的加权组合。

#### 2.3.6 外部记忆架构

- **NTM（2014）**与 **DNC（2016）**：开创了可微外部记忆，支持内容寻址、动态分配、时间链接矩阵。
- **MemoryLLM（2024）**：在 Transformer 层中嵌入固定大小记忆池，通过自注意力融合记忆与当前激活，更新采用随机丢弃 K 槽位 + 追加新压缩标记，遵循指数遗忘曲线。

**对 Mimir 的启示**：我们的定长原型矩阵 + 指数衰减 + 容量淘汰策略与 MemoryLLM 一致，但更简单（无反向传播更新记忆池，只用局部 Hebbian 规则）。

## 四、模块边界定义

| 模块 | 核心职责 | 对外接口语义 |
|---|---|---|
| **PrototypeStore** | 管理原型矩阵的生命周期：查找、激活、更新、淘汰 | `lookup(emb) → proto_id`<br>`update(proto_id, delta)`<br>`prune(k) → removed_ids` |
| **InferencePipeline** | 纯计算路径：base embedding + 原型残差调制 + 预测误差 | `encode(texts) → embeddings` |
| **PredictionPolicy** | 根据历史原型激活预测下一个原型 | `predict(context) → predicted_activation`<br>`compute_surprise(actual, predicted) → score` |
| **LearningPipeline** | 异步消费推理事件，批量更新原型与转移矩阵 | `learn(batch) → LearnReport` |
| **EventBus** | 解耦推理与训练，支持背压与重放 | `publish(event)`<br>`subscribe(handler)` |
| **RetentionPolicy** | 强度衰减与淘汰策略（可插拔） | `decay(meta, steps) → meta`<br>`select_for_eviction(pool, n) → ids` |
| **Retrieval** | 混合检索策略：向量 cosine、BM25、RRF 融合 | `score(query, memories) → scores`<br>`fuse(rankings) → scores` |
| **Lifecycle** | 记忆生命周期元数据与重排：recency、importance、access、stale | `score(memories) → scores`<br>`mark_stale(...) → ids` |
| **Persistence** | 系统状态的检查点与恢复 | `save(state, path)`<br>`load(path) → state` |

## 五、关键设计决策

| 决策 | 理由 |
|---|---|
| 推理管道不写任何状态 | 保证 `encode()` 延迟可预测，支持高并发，无需锁 |
| `encode()` 可返回 `surprise_score` | 让调用方判断"这段输入是否值得记忆" |
| 学习通过 EventBus 异步执行 | 调用方不阻塞；可批量聚合；天然支持背压 |
| 预测编码采用 1 阶马尔可夫转移矩阵 | 足够捕捉话题单步跳转；避免 N-gram 维度爆炸 |
| 转移矩阵全局共享 | 话题转移规律跨 workspace 共通；分矩阵无法解决冷启动 |
| surprise 阈值 = 自适应基线 × 可调 multiplier | 系统自己算基线，调用方按场景调整敏感度 |
| 原型矩阵容量固定（如 1024） | 内存占用可预测，遗忘策略简单明确 |
| 不引入多头/子空间结构 | 避免不必要的调试复杂度；单一矩阵已足够表达概念偏差 |
| 持久化只存矩阵 + 元数据 | 慢权重按名称重建，状态体积极小，天然增量检查点 |
| 原型用 softmax 加权激活（P0），P1 迁移到 top-k 稀疏 | P0 保证可微稳定；P1 引入 HiCL 式 DG 稀疏编码，提升效率与可解释性 |
| LearningPolicy 插件化，P0 实现 Oja，P2 预留 BCM | Oja 稳定基线；BCM 提供更生物合理的双向可塑性 |
| 学习率按访问次数衰减 | 模拟"熟悉后收益递减"，防止过拟合高频输入 |
| 遗忘同时做强度衰减与容量淘汰 | 双重保险：时间衰减负责自然遗忘，容量淘汰负责硬边界 |

## 六、边界覆盖场景

| 场景 | 处理策略 |
|---|---|
| **冷启动** | 原型矩阵为空，`encode()` 退化回 base embedding，surprise_score 固定为 1.0 |
| **容量满** | 按 `strength × frequency` 排序，淘汰末位 10% 或固定数量 |
| **罕见/异常输入** | 与所有原型最大相似度低于阈值时，触发快速原型初始化；surprise 同步升高 |
| **预测命中** | 实际激活与预测一致 → 强化对应转移矩阵元素 |
| **预测错误** | 实际激活与预测偏离 → 弱化旧转移 + 视情况创建新原型 |
| **高并发调用** | 推理路径只读，无锁；训练路径单 worker 串行消费 |
| **版本迁移** | 矩阵格式保持稳定；慢权重模型名作为配置，加载时按名重建 |
| **长文本** | 由慢权重模型负责截断/分块；Mimir 本身只处理固定维度向量 |
| **空输入** | 返回零向量，不触发学习事件，记录 warning |

## 七、产品增强路线图

基于竞品分析（见 `.note/comparison.md`），Mimir 应在保持"可塑性"差异化的前提下，系统性地补齐工程与产品短板。优先级如下：

### P1：混合检索与生命周期管理（当前迭代）

**目标**：补齐 recall 的关键词匹配短板，并引入记忆质量与衰减机制。

- **BM25 + 向量混合检索**
  - 新增 `mimir/infrastructure/retrieval/` 模块
  - 抽象 `MemoryScorer`、`FusionStrategy` 协议
  - 实现 `VectorScorer`（现有 cosine）、`BM25Scorer`、`RRFFusion`
  - `InMemoryAgentAdapter.recall()` 默认开启混合检索，保留纯向量回退
- **记忆生命周期元数据**
  - `Memory` 新增 `importance`、`access_count`、`last_accessed_at`
  - 新增 `mimir/infrastructure/lifecycle/` 模块
  - 实现 `LifecycleScorer`：综合 recency、importance、access_count 重新排序
- **去重与 stale 标记**
  - 基于内容 hash 的去重
  - 长时间未访问且低重要性的记忆标记为 stale（不删除，仅降权）

**验收标准**：
- recall 在混合关键词+语义 query 上召回率高于纯向量
- 新增单元测试覆盖 BM25、RRF、去重、stale 场景
- 150+ 测试全部通过

### P2：质量门控与集成体验

- **记忆质量门控**
  - 重复阻断、矛盾提示、HITL preview
- **`mimir setup <agent>` 一键配置**
  - 自动生成 Kimi Code / Claude Code / Codex / OpenCode 配置
- **Secret redaction**
  - hook 存储前对密钥/API key 脱敏
- **Project Context Discovery**
  - 自动扫描 `CLAUDE.md` / `AGENTS.md` / `.cursorrules` 注入上下文

### P3：异步化与可观测性

- **Embedding 异步队列**
  - `store` 不再阻塞等待 embedding backend
- **LongMemEval 评测 harness**
  - 把我们的 recall 跑在公开 benchmark 上
- **原型可解释性工具**
  - 查看每个原型代表的文本
- **Metrics / Telemetry**
  - Prometheus / OpenTelemetry 指标

### P4：高级演进

- 慢权重蒸馏：将高频原型模式通过 LoRA 写回 base 模型
- 基于 surprise 的优先重放
- BCM 双向可塑性
- 多租户隔离
- 分布式训练

## 八、成功指标

| 指标 | 目标 |
|---|---|
| 推理延迟增量 | `< 20%` 相比 base 模型 |
| 状态文件大小 | `< 100MB`（k=1024, dim=768） |
| 冷启动行为 | 与 base 模型一致 |
| 学习收敛 | 同一领域输入 10 次后语义相似度提升 > 10% |
| 预测准确率 | 连续对话中，下一步原型预测 Top-1 准确率 > 40% |
| 惊喜区分度 | 新话题 surprise_score 比熟悉话题高 2 倍以上 |
| 遗忘稳定性 | 容量满后旧概念 recall 平滑下降，无抖动 |
| 混合检索 recall@5 | 关键词+语义 query 上比纯向量提升 > 15% |

## 九、风险与应对

| 风险 | 可能性 | 应对 |
|---|---|---|
| 单一矩阵表达能力不足 | 中 | 先固定 k=1024 验证；不足时引入分层原型 |
| 异步学习导致状态延迟 | 中 | EventBus 可配置同步模式用于关键学习 |
| 原型坍缩（多个原型趋同） | 低 | 更新时加入正交正则；定期合并相似原型 |
| 转移矩阵稀疏/发散 | 低 | 对转移矩阵做 L1/L2 正则；定期归一化行和 |
| 慢权重模型升级后矩阵失效 | 低 | 版本号标记；升级时可选清空或迁移 |
| BM25 实现引入依赖 | 低 | 默认纯 Python，可选 `rank-bm25` |
| 生命周期参数难调 | 中 | 提供 status 暴露指标，通过 eval 校准 |
